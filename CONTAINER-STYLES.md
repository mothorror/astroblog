# 容器区块样式使用指南

## 概述

BlackWhite 主题现在支持 8 种预定义的容器区块类型，用于组织和突出显示特定类型的内容。

## 支持的容器类型

| 类型 | 语法 | 边框颜色 | 背景颜色 | 用途 |
|------|------|----------|----------|------|
| **info** | `:::info` / 内容 / `:::` | 蓝色 (#5bc0de) | 浅蓝 (#e3f2fd) | 信息提示 |
| **note** | `:::note` / 内容 / `:::` | 深灰 (#6c757d) | 浅灰 (#f8f9fa) | 笔记记录 |
| **warning** | `:::warning` / 内容 / `:::` | 橘色 (#ff9800) | 浅橘 (#fff3e0) | 警告提醒 |
| **danger** | `:::danger` / 内容 / `:::` | 红色 (#dc3545) | 浅红 (#ffebee) | 危险警示 |
| **tip** | `:::tip` / 内容 / `:::` | 绿色 (#4caf50) | 浅青绿 (#e8f5e9) | 技巧建议 |
| **success** | `:::success` / 内容 / `:::` | 深绿 (#28a745) | 浅绿 (#d4edda) | 成功状态 |
| **primary** | `:::primary` / 内容 / `:::` | 深蓝 (#34495e) | 浅灰蓝 (#eaedf0) | 主要内容 |
| **flat** | `:::flat` / 内容 / `:::` | 灰色 (#95a5a6) | 浅灰 (#f5f5f5) | 扁平样式 |

## 语法格式

### 基本语法

```markdown
:::类型
内容
:::
```

### 带标题的容器

```markdown
:::info
### 标题
这是信息提示框的内容，可以包含标题、列表、代码块等。
:::
```

### 嵌套内容

```markdown
:::warning
### 注意事项

- 项目1
- 项目2
- 项目3

**重要提示**：请仔细阅读以上内容。
:::
```

### 包含代码块

```markdown
:::info
### 示例代码

```javascript
function hello() {
  console.log('Hello!');
}
```
:::
```

## 样式特点

### 设计特性

- **统一边框**：8px 左侧彩色边框（所有类型统一）
- **无边框圆角**：0px 圆角（扁平化设计）
- **柔和背景**：对应颜色的浅色背景
- **统一间距**：32px 上下外边距，12px 内边距
- **阴影效果**：轻微阴影提升层次感
- **响应式**：自动适应不同屏幕尺寸

### 排版特性

- **适当间距**：16px 上下外边距，12px 内边距
- **行高优化**：1.6 倍行高保证可读性
- **元素重置**：容器内第一个和最后一个元素的边距自动调整

## 使用示例

### 示例 1：信息提示

```markdown
:::info
本文档介绍了 BlackWhite 主题的容器区块功能。容器区块可以帮助您更好地组织和展示内容。
:::
```

### 示例 2：警告提醒

```markdown
:::warning
**注意**：修改配置文件后需要重新构建才能生效！

运行以下命令重新构建：
```bash
npm run build
```
:::
```

### 示例 3：成功状态

```markdown
:::success
✅ 构建成功！

您的博客已准备就绪，可以开始发布内容了。
:::
```

### 示例 4：技巧提示

```markdown
:::tip
**技巧**：使用容器区块可以让重要信息更加突出。

尝试在不同的场景中使用不同的容器类型：
- info：补充说明
- warning：注意事项
- tip：实用技巧
:::
```

## 最佳实践

1. **选择合适的类型**
   - 使用 `info` 提供补充信息
   - 使用 `warning` 标记注意事项
   - 使用 `danger` 警示严重错误
   - 使用 `tip` 分享实用技巧
   - 使用 `success` 确认操作成功

2. **保持简洁**
   - 容器内容应简明扼要
   - 避免在容器内嵌套过多层级
   - 使用标题和列表增强可读性

3. **视觉层次**
   - 不要过度使用容器区块
   - 在长文档中合理使用不同类型区分内容
   - 保持页面整体视觉平衡

## 技术实现

### Remark 插件

容器区块通过 `remark-directive` 和自定义 `remark-custom-blockquotes.js` 插件实现。

### CSS 样式

样式定义在以下位置：
- **全局样式**：`src/layouts/NextLayout.astro`
- **正文页面**：`src/pages/blog/[...slug].astro`
- **卡片组件**：`src/components/PostCardNext.astro`

### HTML 输出

容器区块会被转换为带有特定类名的 blockquote 元素：

```html
<blockquote class="info">
  <p>内容</p>
</blockquote>
```

## 常见问题

### Q: 为什么容器不显示？

A: 请确保使用正确的语法（三重冒号 `:::`）和有效的容器类型名称。

### Q: 可以自定义容器颜色吗？

A: 当前版本使用预定义的颜色方案。如需自定义，请修改 CSS 文件中的颜色值。

### Q: 容器可以嵌套吗？

A: 不建议在容器内嵌套另一个容器，这可能导致样式冲突。

### Q: 容器内可以包含图片吗？

A: 可以，容器支持所有 Markdown 语法，包括图片、链接、代码块等。
